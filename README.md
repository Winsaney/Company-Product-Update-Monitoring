# 开源伙伴监控中心

一款轻量级的 **GitHub 仓库版本监控与 AI 智能分析平台**，专为开发者和技术团队设计。实时监控开源项目及上下游依赖，通过 AI 自动生成版本总结和每周动态周报。

## ✨ 核心特性

### 📦 仓库监控
- **批量监控**：统一管理多个 GitHub 仓库，实时追踪版本更新
- **智能检测**：自动轮询 GitHub API，检测新版本发布
- **历史记录**：完整的版本更新历史，支持现有版本查看

### 🤖 AI 智能分析
- **单仓库总结**：一键生成 Release Notes 的中文精简总结
- **每周动态周报**：汇总所有监控仓库一周内的版本发布动态
- **多模型支持**：兼容 OpenAI、DeepSeek、通义千问等任意 OpenAI 规范接口

### 🎨 现代化界面
- **精美 UI 设计**：受 Linear/Vercel 启发的现代化深色主题
- **浅色/深色模式**：支持一键切换，自动记忆用户偏好
- **响应式布局**：完美适配桌面和移动设备

### 🔒 安全可靠
- **JWT 认证**：全站受密码保护，支持环境变量自定义
- **数据持久化**：支持 Railway Volume 挂载，更新不丢失
- **本地存储**：轻量级文件存储，无需数据库依赖

### 📧 多渠道通知
- **邮件通知**：新版本自动发送精美 HTML 邮件
- **即时提醒**：支持扩展到 Slack、钉钉等即时通讯平台

## 🚀 快速上手 (本地运行)

**1. 安装依赖**
```bash
npm install
```

**2. 启动服务**
```bash
npm start
```
服务默认运行在 `http://localhost:3000`。初始登录密码为 `admin123`（在没有设置环境变量的情况下）。

## ☁️ 云端部署 (推荐 Railway)

本项目针对类似 Railway 的 Serverless/Dockerless 云容器平台做了特别优化。

### 部署步骤
1. 将当前代码库 Fork/推送到受支持的 GitHub 仓库。
2. 登录 [Railway](https://railway.app/)，选择 **New Project** -> **Deploy from GitHub repo**，选取该仓库。
3. Railway 会自动使用内部生成的 `PORT` 启动容器引擎。

### 配置环境变量 (Variables)
为了保障应用的安全和数据的持久化存储，强烈建议添加以下环境变量：

- `ADMIN_PASSWORD`：[重要] 修改默认的管理员登录密码（尽量避免空格等特殊空字符）。
- `DATA_DIR`：[重要] 配合持久化存储使用，设定数据文件在容器内保存的位置。

*(环境变量 `JWT_SECRET` 和 `PORT` 框架会自动处理，通常无需手动分配)*

### 🔥 配置防止数据丢失挂载盘 (Persistent Volume)
由于 Railway 每次更新代码都会重启创建全新的容器，你需要为其挂载持久化硬盘以防止设置清除：
1. 去 Railway 项目的 **Settings** -> **Volumes**，点击 **Add Volume** 挂载卷。
2. 挂载路径（Mount Path）推荐填写：`/app/data`。
3. 接着去 **Variables** 环境变量里，新增一条 `DATA_DIR=/app/data`。
4. 发生下次部署后，你的所有记录和配置便会永久安全地保存在这个卷上！

## ⚙️ 功能说明

### 📊 仪表盘
- 监控仓库统计概览
- 最新版本更新列表
- 系统状态一目了然

### 📁 仓库管理
- 添加/删除监控仓库
- 实时查看仓库状态
- 检查最新版本信息
- 一键触发手动检查

### 📜 更新历史
- 查看所有检测到的版本更新
- 区分"新发布"和"现有版本"
- 完整的时间线追踪

### 🤖 AI 总结
- **单仓库总结**：为任意仓库生成 AI 版本摘要
- **每周动态周报**：一键生成所有仓库的综合性周报
- **历史记录**：查看过往生成的所有总结
- 支持重新生成和对比

### ⚙️ 设置
- **GitHub Token**：配置 Personal Access Token 提高请求限额
- **检查间隔**：设置自动检查频率（默认 7 天）
- **邮件服务**：配置 SMTP 服务器和收件人
- **AI 模型**：配置 OpenAI 兼容的 API 地址和密钥
- **主题切换**：浅色/深色模式自由切换

## 📝 技术栈

- **后端**：Node.js + Express
- **认证**：JWT (jsonwebtoken)
- **任务调度**：node-cron
- **HTTP 客户端**：node-fetch
- **存储**：轻量级本地 JSON 文件存储
- **前端**：原生 HTML5 + CSS3
  - 无框架依赖，极致加载速度
  - CSS Variables 实现主题切换
  - 响应式设计，支持移动端

## 🎯 使用场景

- **技术团队**：监控依赖库的版本更新，及时获取安全补丁
- **产品经理**：跟踪竞品的功能迭代和发布节奏
- **开源贡献者**：关注关注项目的动态
- **企业用户**：统一管理内部使用的开源组件
